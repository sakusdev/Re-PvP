# Fix Notes - 2026-07-05

This pass addresses issues found during the first grounded code review.

## Fixed

### 1. Local player lookup no longer scans every frame

Updated:

```text
src/RePvP/UnityLocalPlayerResolver.cs
```

Changes:

- Added local player candidate caching.
- Added retry throttling.
- Scene-wide `FindObjectsOfType<GameObject>()` now runs only when the cache is missing/stale and the retry interval has elapsed.

### 2. Extraction result reporting is now accurate

Updated:

```text
src/RePvP/RoundManager.cs
```

Changes:

- `TryMarkHeisterExtracted(GameObject)` now returns `true` only when extraction actually succeeds.
- `MarkHeisterExtracted(PlayerRef)` now returns `bool`.
- Duplicate extraction attempts return `false`.
- Non-Heister or wrong-phase extraction attempts return `false`.

### 3. Alarm phase is guarded

Updated:

```text
src/RePvP/RoundManager.cs
```

Changes:

- `TriggerAlarm()` now only works during `Preparation` or `Heist`.
- Debug F8 can no longer push `WaitingForPlayers`, `RoundEnd`, or other invalid phases into Alarm/Extraction.

### 4. Tentative Harmony patches are disabled by default

Updated:

```text
src/RePvP/RePvPConfig.cs
```

Changes:

- `EnableHarmonyPatches` now defaults to `false`.
- This prevents unverified reflection patch candidates from accidentally affecting gameplay.

### 5. Harmony argument handling is safer

Updated:

```text
src/RePvP/Patching/PatchReflection.cs
src/RePvP/Patching/ValuableCashInPatch.cs
src/RePvP/Patching/ExtractionPatch.cs
```

Changes:

- Added safe integer argument extraction helpers.
- Replaced fragile `__0` extraction logic with `object[] __args`.
- Extraction patch no longer assumes the target method has at least one argument.
- Valuable cash-in patch now supports separate static and instance postfixes.

### 6. Player discovery avoids unsafe static method patching

Updated:

```text
src/RePvP/Patching/PlayerDiscoveryPatch.cs
```

Changes:

- Skips static candidate methods because they cannot safely provide a player instance through `__instance`.
- Added missing postfix null guard.
